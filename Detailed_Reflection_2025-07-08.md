# 오늘의 리플렉션 (2025-07-08)

## 개요

오늘은 주로 React 학습에 집중하여 컴포넌트, Props, State 개념을 익히고 실제 코드에 적용하는 작업을 진행했습니다. Vite를 사용하여 두 개의 새로운 React 프로젝트를 초기화하고, 컴포넌트 기반 개발 방식에 대한 이해를 높였습니다. 또한, 간단한 머신러닝 코드 정리도 함께 수행했습니다.

## 주요 작업 및 변경 내역

### 1. React `useState` 학습 및 카운터 앱 개선 (f98c53c, f0459dc)

- **`useState`를 사용한 상태 관리**: `useState` 훅을 사용하여 카운터 앱의 상태(`count`)를 관리하는 방법을 학습했습니다.
- **기능 개선**:
    - `onIncrease`와 `onDecrease` 함수를 구현하여 버튼 클릭 시 숫자를 증가시키고 감소시키는 기능을 만들었습니다.
    - `onDecrease` 함수에 **0 이하로 내려가지 않도록** 조건문을 추가하여 사용성을 개선했습니다.
- **코드 정리**:
    - Vite 초기 템플릿에 포함된 불필요한 로고 이미지와 CSS (`App.css`, `index.css`)를 제거하여 코드를 단순화했습��다.
    - `App.jsx`의 구조를 간소화하여 핵심 기능(카운터)에만 집중하도록 수정했습니다.

### 2. React 컴포넌트와 Props 학습 (1211894, 5638b6a, a508c9e, 62f3bd0, 9c4f223)

- **프로젝트 초기화**: `02component_props` 라는 이름으로 새로운 Vite 기반 React 프로젝트를 생성했습니다.
- **컴포넌트 분리 및 재사용**:
    - **`Menubox` 컴포넌트**: 메뉴 정보를 표시하는 재사용 가능한 컴포넌트를 만들었습니다.
    - **`Member` 컴포넌트**: 팀원 정보를 표시하는 컴포넌트를 추가로 생성하여 컴포넌트의 재사용성을 연습했습니다.
- **Props를 이용한 데이터 전달**:
    - 부모 컴포넌트(`App2.jsx`)에서 자식 컴포넌트(`Menubox`, `Member`)로 `name`, `price`, `temp`, `sub`, `mbti` 등의 데이터를 Props로 전달하는 방법을 익혔습니다.
    - 구조 분해 할당을 사용하여 `props` 객체를 더 간결하게 처리했습니다.
- **스타일링**:
    - `App2.css`를 생성하고 메뉴 카드에 대한 스타일(그림자, 호버 효과 등)을 적용했습니다.
    - 호버 시 카드가 위로 떠오르는 효과(`transform: translateY(-10px)`)를 주어 동적인 UI를 구현했습니다.
- **프로젝트 구조 변경**:
    - 초기 `App.jsx`와 `App.css`를 삭제하고, 실습에 더 적합��� `App2.jsx`와 `App2.css`로 대체했습니다.
    - `main.jsx`에서 렌더링할 컴포넌트를 `App2`로 변경하고, 초기 설정에 있던 `StrictMode`를 잠시 제거했습니다.
